# 来源与维护规范

## 1. 来源等级

| 等级 | 来源 | 适合证明 | 注意事项 |
|---|---|---|---|
| A | 政府法令、制造商规格页/目录、钢厂数据表、专利、设计者原文 | 法规原文、SKU 规格、材料成分、发布日期 | 制造商历史仍可能带品牌叙事；保留页面日期/目录年份 |
| B | 行业协会、博物馆、学术论文、专业测试机构 | 行业趋势、历史、可复现实验 | 检查方法、样本量和利益关系 |
| C | 授权经销商、专业媒体、书籍 | 停产规格、价格快照、访谈、补充历史 | 与一手来源交叉验证 |
| D | 论坛、视频、用户测量、二级市场 | 发现线索、主观体验、批次问题 | 不单独用于核心规格或法律结论 |
| E | 聚合站、无作者 SEO 内容、AI 摘要 | 仅作线索 | 不进入已验证事实 |

## 2. 每条声明的最小记录

建议来源对象至少包含：

```json
{
  "claim": "Small Sebenza 31 当前刀刃长度为 76.17 mm",
  "value": 76.17,
  "unit": "mm",
  "scope": "Small Sebenza 31",
  "variant_or_sku": "Plain / current production",
  "source_url": "https://chrisreeve.com/pages/faq",
  "source_type": "manufacturer",
  "source_title": "Chris Reeve Knives FAQ",
  "accessed_at": "2026-07-04",
  "effective_at": "2026-07-04",
  "quote_or_locator": "Sebenza 31 > Technical Specifications > BLADE",
  "status": "verified"
}
```

## 3. 事实状态

- `verified`：一手来源直接支持，范围与版本明确。
- `corroborated`：两个相互独立的可靠二手来源一致，一手资料不可得。
- `manufacturer_claim`：制造商主张但缺乏独立验证，如销量或“最强”。
- `approximate`：因版本、手工差异或单位换算只能近似。
- `conflicted`：可靠来源冲突，页面应展示分歧而非任选其一。
- `stale`：超过更新周期或型号已变更，等待复核。

## 4. 数值规则

1. 保存来源原始单位与原始值，另存 SI 换算值；不要反复换算。
2. 换算值精度不高于来源。`3.24 in` 可显示约 `82.3 mm`，不可伪装成制造公差级精度。
3. 手工产品、不同柄材和官方“约”值保留 `approximate=true`。
4. 刃长与切削刃长、总长与闭合长、重量与含鞘重量分开。
5. 价格必须带币种、含税口径、市场、SKU 和日期。

## 5. 版本和更新

- 当前量产 SKU：每 6 个月自动检查链接和页面哈希，至少每年人工复核。
- 停产/历史款：初次双源验证后，来源失效时转存网页存档线索，不静默改值。
- 法规：至少每季度检查；页面显示最后复核日期，并在变更时保留旧规则有效期。
- 市场数据：每年更新；不同研究机构口径不能拼接成时间序列。

## 6. 编辑检查清单

- [ ] 这是型号家族还是具体 SKU？
- [ ] 规格适用于哪个时间和市场？
- [ ] 每个核心字段是否有能直接证明它的来源？
- [ ] 是否把制造商主张写成独立事实？
- [ ] 是否混淆钢材理论性能与成品实测？
- [ ] 是否错误地给出跨司法辖区“合法”结论？
- [ ] 是否标出近似、冲突、缺失和版本差异？
- [ ] 中英文名、别名、单位和锁型术语是否规范？
- [ ] 链接可访问，且不是搜索结果页？
- [ ] 内容是否保持工具、设计、历史和安全导向？

## 7. 本研究的已知限制

- 未购买付费零售面板或海关明细，无法给出可审计的小刀细分市场规模/份额。
- 部分制造商页面为动态页面，搜索引擎索引可能落后于当前 SKU。
- 未对实物做卡尺、称重、硬度或切割测试；所有规格均为资料研究。
- 仅用中国、英国、美国联邦层面示范法规复杂度，未覆盖州、省、市、运输场所和平台政策。
- “知名”样本是为设计史与产品谱系服务的编辑选择，不是销售或性能排名。
